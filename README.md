# 🚴 Bike Arena — Application mobile de défis cyclistes

Application mobile expérimentale développée en React Native visant à combiner cartographie, création de trajets vélo et défis entre cyclistes, autour d’un environnement inspiré d’un “terrain de jeu” sportif.

---

## 🔎 Aperçu du projet

**Bike Arena** est une application mobile conçue comme un équivalent orienté cyclisme d’un outil de navigation type Google Maps, basé sur OpenStreetMap.

L’objectif est de permettre aux utilisateurs de :

- créer leurs trajets à vélo,
- retrouver des parcours adaptés à leur pratique,
- défier des amis via l’application,
- centraliser des services utiles aux cyclistes.

Ce projet s’inscrit dans une démarche d’apprentissage du développement mobile tout en restant dans l’écosystème JavaScript.

---

## 🎯 Contexte et objectifs

Ce projet est né d’un contexte personnel : utilisant le vélo quotidiennement (trajets école, déplacements, pratique en club de compétition), j’ai souhaité relier deux passions :

- le **cyclisme**
- le **développement logiciel**

L’idée était de créer un espace — *Bike Arena* — où chaque cycliste pourrait trouver des défis adaptés à son niveau et organiser ses trajets.

### Objectifs initiaux

- Apprendre le développement mobile avec **React Native**
- Créer un système de création de trajets basé sur OpenStreetMap
- Permettre des défis entre utilisateurs invités

### Objectifs secondaires

- Intégrer des services utiles aux cyclistes :
  - choix du type de route
  - recherche de réparateurs vélo
  - services liés à la mobilité douce

## 🧠 Challenges & apprentissages

### Apprentissage d’une nouvelle technologie

Le principal objectif était de découvrir **React Native** et le développement mobile tout en restant dans une stack JavaScript connue.

Un constat important issu du projet :

> L’utilisation intensive de l’IA accélère fortement la production initiale, mais le temps nécessaire pour comprendre en profondeur le code généré peut être significativement plus long.

Cette expérience a mis en évidence l’importance :
- de comprendre avant d’accélérer,
- d’ancrer les apprentissages via la documentation officielle.

---

## 🗺️ Roadmap

Évolutions envisagées :

- Implémenter les fonctionnalités prévues :
  - création de trajets
  - défis entre utilisateurs
  - filtres de routes
  - services cyclistes
- Adopter une approche **test-first** :
  - génération de tests en amont
  - implémentation progressive des fonctionnalités
- Reprendre l’apprentissage via la documentation officielle React Native, avec une progression plus incrémentale et moins assistée par IA.
